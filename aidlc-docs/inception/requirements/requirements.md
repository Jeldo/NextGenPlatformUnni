## Functional Requirements

### FR-1: 시술 자동 기입
- 예약 확정(FIXED) 이벤트 수신 시 해당 시술을 고객 캘린더에 자동 기입
- 수술은 제외, 시술만 대상
- 이벤트 기반 아키텍처 (메시지 큐/이벤트 버스 구독)
- **이벤트 모킹 전략**:
  - 프로덕션: AWS SQS/SNS 구독
  - 개발/테스트: 인메모리 채널 + Mock HTTP 엔드포인트 (`/mock/events/reservation-fixed`)로 이벤트 수동 발행
  - 동일 인터페이스로 환경 전환 (의존성 주입)

### FR-2: 자동 기입 시술 수정/삭제
- 자동 기입된 시술의 날짜 수정 가능
- 자동 기입된 시술 삭제 가능 (실제로 받지 않은 경우)
- **시술 정보 수정 플로우**:
  - [받은 시술 정보 수정] 버튼 → 수정 페이지 이동
  - 3단계 드롭다운 (바텀시트 방식):
    1. 시술 카테고리 (예: 보톡스, 필러, 레이저)
    2. 정확한 시술명 (예: 사각턱, 이마, 스킨 보톡스) — 카테고리에 따라 필터링
    3. 용량:
       - 숫자 인풋 + 단위 드롭다운 (cc, ml 등)
  - 각 드롭다운 클릭 시 바텀시트로 선택
- **참고**: 같은 날짜에 여러 시술 기록을 추가할 수 있음 (수동 추가로 처리)
- **참고**: 3단계 드롭다운을 위한 "시술 데이터" 구축 필요 (MVP 포함)

### FR-3: 시술 수동 CRUD
- **시술 추가 플로우** ([시술 추가하기] 버튼):
  1. 날짜 선택
  2. 병원명 입력 (자유 텍스트)
  3. 시술 정보 입력 — 3단계 드롭다운 (FR-2와 동일)
  4. 추가 메모 입력 (선택)
- 기입된 시술 정보 수정 (본인 캘린더에만 반영)
- 기입된 시술 삭제 (본인 캘린더에만 반영)

### FR-4: 다음 시술 예정일
- 시술별 추천 주기(룰베이스) 기반으로 다음 예정일 계산 및 표시
- 예정일은 확정 일정과 구분되는 UI로 표시 (점선 등)

### FR-5: 리마인드 알림
- 시술 주기 도래 시 푸시 알림 발송 요청
- 기존 푸시 알림 시스템에 알림 요청 전송
- 개발 환경에서는 알림 발송/수신 모킹 제공

### FR-6: 시술 통계
- 캘린더 메인 페이지 하단에 인라인 표시 (별도 페이지 아님)
- 캘린더 등록 시술 기준 시술별 시행 횟수 집계
- 예: 보톡스 n회, 필러 n회

### FR-7: 구글 캘린더 연동
- 시술 상세 페이지 하단에 outline 스타일 버튼 "구글 캘린더에 등록하기"
- 클라이언트에서 Google Calendar API 직접 호출
- 특정 시술 일정을 구글 캘린더에 단건 등록
- 다음 예정 시술도 별도로 구글 캘린더에 등록 가능
- Google OAuth 인증 필요

### FR-8: 추천 주기 관리 (관리자 — 독립 서비스)
- 시술 카테고리별 고정 주기를 DB에 저장
- FastAPI 기반 독립 관리자 API로 주기 데이터 CRUD
- 캘린더 서비스에서 내부 API 호출로 주기 데이터 조회

### FR-9: 시술 데이터 관리 (MVP 포함)
- 3단계 드롭다운을 위한 시술 마스터 데이터 구축
- 카테고리 → 시술명 → 용량 단위 계층 구조
- 관리자 API로 시술 데이터 CRUD (FR-8과 동일 FastAPI 서비스에서 관리)

## Non-Functional Requirements

### NFR-1: 성능
- 중규모 사용자 대응 (DAU 1만~10만)
- 캘린더 조회 API 응답 시간 < 200ms (p95)

### NFR-2: 보안 (최소 수준)
- 입력값 검증 (validation)
- SQL injection 방지 (parameterized queries)
- 인증/인가는 이번 범위에서 제외

### NFR-3: 테스트
- Classicist testing 방식 (실제 객체 우선, 외부 의존성만 모킹)
- 단위 테스트 필수
- 날짜 계산, 주기 로직 등 순수 함수에 Property-Based Testing 적용
- 외부 의존성(이벤트, 알림, FastAPI 서비스)만 모킹

### NFR-4: 확장성
- 이벤트 기반 아키텍처로 느슨한 결합
- Phase 2 (AI 추천 주기) 확장 고려한 설계

---

## Out of Scope

- 수술 기록
- 아카이브 기능 (남은 시술권 관리, 후기 연동, 다회권 리마인더)
- 강남언니 외부 병원 예약 정보 직접 연동
- AI/LLM 기반 추천 주기 (Phase 2)
- 구글 캘린더 → 강남언니 캘린더 역방향 동기화
- 인증/인가 시스템

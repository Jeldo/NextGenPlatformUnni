# User Stories — 시술 관리 캘린더

> 형식: Epic → Story 계층 | User Journey 순서 | Given-When-Then AC
> 페르소나: 고객(민지), 관리자(운영팀)

---

## Epic 1: 시술 자동 기입

> 강남언니에서 예약 확정된 시술이 캘린더에 자동으로 기록된다.

### Story 1.1: 예약 확정 시 캘린더 자동 등록
**As a** 고객  
**I want** 강남언니에서 예약 확정(FIXED)된 시술이 캘린더에 자동 기입되길  
**So that** 별도 입력 없이 시술 이력이 관리된다

**Acceptance Criteria:**
- Given 고객이 강남언니에서 시술을 예약 확정했을 때
- When 예약 확정(FIXED) 이벤트가 발생하면
- Then 해당 시술이 고객의 캘린더에 자동 기입된다
- And 시술명, 날짜, 병원명이 포함된다
- And 수술은 제외되고 시술만 기입된다

### Story 1.2: 자동 기입된 시술 날짜 수정
**As a** 고객  
**I want** 자동 기입된 시술의 날짜를 수정할 수 있길  
**So that** 실제 시술 받은 날짜와 다를 경우 보정할 수 있다

**Acceptance Criteria:**
- Given 자동 기입된 시술 기록이 있을 때
- When 날짜 수정을 요청하면
- Then 해당 기록의 날짜가 변경된다
- And 변경된 날짜 기준으로 다음 예정일이 재계산된다
- And 수정 시 메모도 추가/변경할 수 있다

### Story 1.3: 자동 기입된 시술 삭제
**As a** 고객  
**I want** 자동 기입된 시술을 삭제할 수 있길  
**So that** 실제로 받지 않은 시술을 제거할 수 있다

**Acceptance Criteria:**
- Given 자동 기입된 시술 기록이 있을 때
- When 삭제를 요청하면
- Then 해당 기록이 캘린더에서 제거된다
- And 해당 시술의 다음 예정일도 제거된다

### Story 1.4: 자동 기입된 시술 정보 수정
**As a** 고객  
**I want** 자동 기입된 시술의 시술 정보를 변경할 수 있길  
**So that** 실제 받은 시술과 다를 경우 보정할 수 있다

**Acceptance Criteria:**
- Given 자동 기입된 시술 기록이 있을 때
- When [받은 시술 정보 수정] 버튼을 클릭하면
- Then 수정 페이지로 이동한다
- And 기존 시술 정보가 3단계 드롭다운에 입력되어 있다
- When 시술 카테고리 드롭다운을 클릭하면
- Then 바텀시트가 올라오며 카테고리 목록이 표시된다
- When 카테고리를 선택하면
- Then 두 번째 드롭다운(시술명)이 해당 카테고리로 필터링된다
- When 시술명을 선택하면
- Then 세 번째 영역에서 용량 입력 필드가 표시된다 (숫자 인풋 + 단위 드롭다운)
- When 용량을 입력하면
- Then 숫자 인풋 필드가 생성되어 값을 입력할 수 있다

## Epic 2: 시술 수동 기록

> 외부에서 받은 시술을 직접 캘린더에 기록한다.

### Story 2.1: 시술 직접 추가
**As a** 고객  
**I want** 외부에서 받은 시술을 캘린더에 직접 추가할 수 있길  
**So that** 강남언니 외 병원에서 받은 시술도 한 곳에서 관리된다

**Acceptance Criteria:**
- Given 캘린더 화면에서
- When [시술 추가하기] 버튼을 클릭하면
- Then 시술 추가 페이지로 이동한다
- And 다음 순서로 입력한다:
  1. 날짜 선택
  2. 병원명 입력 (자유 텍스트)
  3. 시술 정보 (3단계 드롭다운)
  4. 추가 메모 (선택)
- When 모든 필수 정보를 입력하고 저장하면
- Then 캘린더에 시술이 등록된다

### Story 2.2: 수동 기입 시술 수정
**As a** 고객  
**I want** 수동으로 기입한 시술 정보를 수정할 수 있길  
**So that** 잘못 입력한 정보를 바로잡을 수 있다

**Acceptance Criteria:**
- Given 수동 기입된 시술 기록이 있을 때
- When 수정을 요청하면
- Then 모든 필드(날짜, 병원명, 시술 정보, 메모)를 수정할 수 있다
- And 수정은 본인 캘린더에만 반영된다

### Story 2.3: 수동 기입 시술 삭제
**As a** 고객  
**I want** 수동으로 기입한 시술을 삭제할 수 있길  
**So that** 불필요한 기록을 제거할 수 있다

**Acceptance Criteria:**
- Given 수동 기입된 시술 기록이 있을 때
- When 삭제를 요청하면
- Then 해당 기록이 캘린더에서 제거된다
- And 삭제는 본인 캘린더에만 반영된다

## Epic 3: 다음 시술 예정일

> 추천 주기에 따라 다음 시술 예정일을 확인한다.

### Story 3.1: 예정일 자동 계산 및 표시
**As a** 고객  
**I want** 각 시술의 다음 예정일을 캘린더에서 확인할 수 있길  
**So that** 다음 시술 시기를 놓치지 않는다

**Acceptance Criteria:**
- Given 시술 기록이 있고 해당 시술에 추천 주기가 설정되어 있을 때
- When 캘린더를 조회하면
- Then 마지막 시술일 + 추천 주기 = 다음 예정일이 표시된다
- And 예정일은 확정 일정과 구분되는 UI(점선 등)로 표시된다

### Story 3.2: 시술 정보 변경 시 예정일 재계산
**As a** 고객  
**I want** 시술 날짜를 수정하면 예정일이 자동으로 재계산되길  
**So that** 항상 정확한 예정일을 볼 수 있다

**Acceptance Criteria:**
- Given 시술 기록의 날짜를 수정했을 때
- When 저장하면
- Then 다음 예정일이 새 날짜 기준으로 재계산된다

## Epic 4: 리마인드 알림

> 시술 주기가 다가오면 푸시 알림을 받는다.

### Story 4.1: 주기 도래 시 푸시 알림 수신
**As a** 고객  
**I want** 시술 주기가 다가오면 푸시 알림을 받길  
**So that** 다음 시술을 잊지 않고 예약할 수 있다

**Acceptance Criteria:**
- Given 시술 기록이 있고 추천 주기가 설정되어 있을 때
- When 다음 예정일이 도래하면
- Then 푸시 알림이 발송된다
- And 해당 카테고리의 미래 예약이 없는 경우: "보톡스 맞은 지 3개월이 됐어요. 다음 시술을 예약해볼까요?"
- And 해당 카테고리의 미래 예약이 있는 경우: "보톡스 예약이 n월n일에 있어요"
- And 푸시 알림을 클릭하면 시술 캘린더 페이지로 이동한다

## Epic 5: 시술 통계

> 받은 시술 횟수를 확인한다.

### Story 5.1: 시술별 횟수 조회
**As a** 고객  
**I want** 시술별로 받은 횟수를 확인할 수 있길  
**So that** 내 시술 이력을 한눈에 파악할 수 있다

**Acceptance Criteria:**
- Given 캘린더에 시술 기록이 등록되어 있을 때
- When 시술 통계를 조회하면
- Then 시술별 횟수가 표시된다 (예: 보톡스 5회, 필러 3회)
- And 자동 기입 + 수동 기입 모두 포함된다

## Epic 6: 구글 캘린더 연동

> 시술 일정을 구글 캘린더에 내보낸다.

### Story 6.1: 시술 일정 구글 캘린더 등록
**As a** 고객  
**I want** 특정 시술 일정을 구글 캘린더에 등록할 수 있길  
**So that** 개인 캘린더에서도 시술 일정을 확인할 수 있다

**Acceptance Criteria:**
- Given 캘린더에 시술 기록 또는 예정일이 있을 때
- When 구글 캘린더 내보내기를 요청하면
- Then Google OAuth 인증을 거쳐 해당 일정이 구글 캘린더에 단건 등록된다

### Story 6.2: 예정 시술 구글 캘린더 등록
**As a** 고객  
**I want** 다음 예정 시술도 구글 캘린더에 등록할 수 있길  
**So that** 예정된 시술 시기를 개인 캘린더에서 리마인드 받을 수 있다

**Acceptance Criteria:**
- Given 다음 시술 예정일이 계산되어 있을 때
- When 해당 예정일을 구글 캘린더에 등록 요청하면
- Then 구글 캘린더에 예정일 일정이 등록된다
- And 일정 제목 앞에 "(예정)" prefix가 붙는다 (예: "(예정) 보톡스 - 사각턱")

## Epic 7: 추천 주기 및 시술 데이터 관리 (관리자)

> 운영팀이 시술 데이터와 추천 주기를 관리한다.

### Story 7.1: 추천 주기 CRUD
**As a** 관리자  
**I want** 시술 카테고리별 추천 주기를 등록/수정/삭제할 수 있길  
**So that** 고객에게 적절한 시술 주기를 안내할 수 있다

**Acceptance Criteria:**
- Given 관리자 API에 접근했을 때
- When 시술 카테고리와 추천 주기(일 수)를 입력하면
- Then 해당 주기가 저장된다
- And 캘린더 서비스에서 조회 시 반영된다

### Story 7.2: 시술 마스터 데이터 CRUD
**As a** 관리자  
**I want** 시술 카테고리/시술명/용량 단위 데이터를 관리할 수 있길  
**So that** 고객의 3단계 드롭다운에 정확한 데이터가 표시된다

**Acceptance Criteria:**
- Given 관리자 API에 접근했을 때
- When 시술 카테고리를 추가하면
- Then 고객 드롭다운에 해당 카테고리가 노출된다
- When 카테고리 하위에 시술명을 추가하면
- Then 해당 카테고리 선택 시 시술명 목록에 노출된다
- When 시술명 하위에 용량 단위를 설정하면
- Then 해당 시술명 선택 시 용량 입력이 가능해진다

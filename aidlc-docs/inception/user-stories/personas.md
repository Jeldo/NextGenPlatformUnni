# Personas — 시술 관리 캘린더

## Persona 1: 고객 (시술 받는 사용자)

| 항목 | 설명 |
|------|------|
| **이름** | 민지 (대표 페르소나) |
| **역할** | 주기적으로 시술을 받는 강남언니 앱 사용자 |
| **목표** | 시술 이력을 한 곳에서 관리하고, 다음 시술 시기를 놓치지 않기 |
| **행동 패턴** | 보톡스/필러/레이저 등을 3~6개월 주기로 반복, 여러 병원 이용 |
| **Pain Point** | 시술 이력을 개인 메모나 기억에 의존, 주기를 놓쳐서 효과 감소 |
| **기대** | 앱에서 자동으로 기록되고, 다음 시술 시기를 알려주길 원함 |

## Persona 2: 관리자 (내부 운영팀)

| 항목 | 설명 |
|------|------|
| **이름** | 운영팀 담당자 |
| **역할** | 강남언니 내부 직원 — 시술 마스터 데이터와 추천 주기 관리 |
| **목표** | 시술 카테고리/시술명/용량 데이터를 정확하게 유지, 추천 주기를 적절히 설정 |
| **행동 패턴** | 새로운 시술 추가, 기존 주기 조정, 데이터 정합성 확인 |
| **Pain Point** | 시술 종류가 다양하고 변경이 잦아 데이터 관리 부담 |
| **기대** | API를 통해 빠르게 데이터를 추가/수정할 수 있길 원함 |
